# 事件相关

# mouseover / enter / out / leave

事件 触发条件 是否冒泡
mouseenter 鼠标指针穿过被选元素 否
mouseover 鼠标指针穿过被选元素或其子元素、孙元素 是
mouseleave 鼠标指针穿过被选元素 否
mouseout 鼠标指针穿过被选元素或其子元素、孙元素 是

  • mouseenter：当鼠标从元素的边界之外移入元素时，事件被触发；而当鼠标本身在元素边界内时，要触发该事件，必须先将鼠标移出元素外，再次移入才能触发。
  • mouseover：和 mouseenter 不同的是，mouseover 支持事件冒泡。

由于 mouseenter 不支持事件冒泡，导致在一个元素的子元素上进入或离开的时候会触发其 mouseover 和 mouseout 事件，但是却不会触发 mouseenter 和 mouseleave 事件。

给左右两边的 ul 分别添加了 mouseover 和 mouseenter 事件，当鼠标进入左右两边的 ul 时，mouseover 和 mouseenter 事件都触发了，但是当移入各自的子元素 li 时，触发了左边 ul 上的 mouseover 事件，然而右边的 ul 的 mouseenter 事件没有被触发。

造成以上现象本质上是 mouseenter 事件不支持冒泡所致。

# 执行顺序

<script>
  var div = document.getElementsByTagName('div')[0];
  eventBind(div, 'mouseenter', 'mouseenter');
  eventBind(div, 'mouseover', 'mouseover');
  eventBind(div, 'mouseout', 'mouseout');
  eventBind(div, 'mouseleave', 'mouseleave');
  function eventBind(ele, event, output){
    ele.addEventListener('event',function(){
      console.log(output);
    })
  }
</script>

# 触发条件

不论鼠标指针穿过被选元素或其子元素、孙元素，都会触发 mouseover 事件；只有鼠标指针穿过被选元素时，才会触发 mouseenter 事件。

<div>
  <h2>被绑定的<span class="eventType"></span>事件被触发<span class="count">0</span></h2>
</div>
<div>
  <h2>被绑定的<span class="eventType"></span>事件被触发<span class="count">0</span></h2>
</div>
<script>
  var div = document.getElementsByTagName('div');
  function bindEvent(ele, event, output){
    var eventType = ele.getElementsByClassName('eventType')[0];
    var count = ele.getElementsByClassName('count')[0];
    eventType.textContent = event;
    ele.addEventListener(event,function(){
      count.textContent = parseInt(count.textContent)+1;
    })
  }

  bindEvent(div[0], 'mouseover', 'mouseover');
  bindEvent(div[1], 'mouseenter', 'mouseenter');
</script>

只有鼠标指针穿过被选元素时，才会触发 mouseleave 事件；不论鼠标指针穿过被选元素或其子元素、孙元素，都会触发 mouseout 事件。

bindEvent(div[0], 'mouseleave', 'mouseleave');
bindEvent(div[1], 'mouseout', 'mouseout');
