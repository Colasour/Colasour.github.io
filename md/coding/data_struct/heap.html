# 堆

# 概念

数据集合有序，能够为各种操作带来便利。但有些应用并不要求所有数据都是有序的，或者在操作开始之前就变得完全有序。一些应用需要先收集一部分数据，从中选出最大或最小的关键码序列，后续收集更多的数据但始终处理数据中最小或最大关键码的记录，比如优先队列。

堆 (Heap) 是最高效的一种数据结构。堆是一棵完全二叉树。

优先队列：一种特殊的队列，队列中元素出栈的顺序是按照元素的优先权大小，而不是元素入队的先后顺序。

  • 关键码

    假定在数据记录中存在一个能够标识数据记录的数据项，并可依据该数据项堆数据进行组织，则称此数据项为关键码 (key)

  • 最小堆

    父节点的关键码小于等于左右孩子节点的关键码，即所有父节点都小于子节点

  • 最大堆

    父节点的关键码大于等于左右孩子节点的关键码，即所有父节点都大于子节点

# 初始化堆

使用数组来存储一个二叉堆，数组存储节点 i 的左右孩子节点的规律如下：

// 索引下标从 0 开始
parent(i) = (i - 1) / 2
left.child(i) = 2 * i + 1
right.child(i) = 2 * i + 2

声明一个数组：

let arr = [ 53, 17, 78, 9, 45, 65, 87, 23 ]

初始的完全二叉树并不满足一个堆的定义，需要对其进行调整。调整的过程自下而上，先保证子树是一个堆，然后从子树延伸，逐步扩大，直到整棵树调整为堆。

# 调整算法

  • 找到所有的父节点，根据父节点的索引从大到小依次调整；每次调整时，从该父节点向下进行调整，使得这个父节点和它的子孙节点构成一个堆

  • 假设数组的大小为 n，则最后一个父节点的索引是 ⌊n / 2⌋ - 1，根节点的索引是 0。如上图最后一个父节点是 9，索引为 3

  • 在调整子树时，根据调整为最小 / 大堆的需求，比较父节点与孩子节点的关键码：

    • 调整为最小堆时，当父节点的关键码大于孩子节点的关键码时，将两者进行交换，否则不需要调整
    • 调整为最大堆时，当父节点的关键码小于孩子节点的关键码时，将两者进行交换，否则不需要调整

    依次重复上述过程，直到父节点的索引为 0

# 算法实现

function heap(arr) {
  const size = arr.length

  // 初始化堆, i 从最后一个父节点开始调整, 直到父节点索引为 0
  for (let i = Math.floor(size / 2) - 1; i >= 0; i--) {
    heapify(i, size)
  }

  return arr

# Top K 问题

一个非常大的数据集有 n 个整数，求集合中最大的 K 个值。

思路：初始化一个大小为 K 的最小堆，先放入 K 个数，此时堆顶元素最小，集合中剩余的数依次和堆顶元素比较，如果比堆顶元素大，则删除堆顶元素，并放入新元素。全部比较完以后，堆里的元素就是最大的 K 个值。
