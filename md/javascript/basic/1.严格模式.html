# 严格模式

🔗 引自《JavaScript 高级程序设计第三版》 附录B

ECMAScript 5 最早引入“严格模式” (strict mode) 的概念。通过严格模式，可以在函数内部选择进行较为严格的全局或局部的错误条件检测。使用严格模式的好处是可以提早知道代码中存在的错误，及时捕获一些可能导致编程错误的 ECMAScript 行为。

严格模式使得 JavaScript 的语法变得更严格，更多的操作会显式报错。

一般来说，非严格模式下会静默失败的情形，在严格模式下就会抛出错误。

# 启用方法

进入严格模式的标志，是一行字符串 use strict：

'use strict';

老版本的引擎会把它当作一行普通字符串，加以忽略。新版本的引擎就会进入严格模式。

严格模式可以用于整个脚本，也可以只用于单个函数。

# 整个脚本文件

use strict 放在脚本文件的第一行，整个脚本都将以严格模式运行。

如果这行语句不在第一行就无效，整个脚本会以正常模式运行。

严格地说，只要前面不是产生实际运行结果的语句，use strict 可以不在第一行，比如直接跟在一个空的分号后面，或者跟在注释后面。

<script>
  'use strict'
  console.log('这是严格模式')
</script>

<script>
  console.log('这是正常模式')
</script>

如果 use strict 写成下面这样则不起作用，严格模式必须从代码一开始就生效：

<script>
  console.log('这是正常模式')
  'use strict'
</script>

# 单个函数

use strict 放在函数体的第一行，则整个函数以严格模式运行：

function strict() {
  'use strict'
  return '这是严格模式'
}

function strict2() {
  'use strict'
  function f() {
    return '这也是严格模式'
  }
  return f()
}

function notStrict() {
  return '这是正常模式'
}

❗ 注意

合并不同的脚本在一个文件时，如果两个脚本的模式不一致，它们的合并可能出错：

  • 严格模式的脚本在前，合并后的脚本都是严格模式
  • 正常模式的脚本在前，合并后的脚本都是正常模式

这两种情况下，合并后的结果都是不正确的。此时可以考虑把整个脚本文件放在一个立即执行的匿名函数之中：

(function () {
  'use strict'
  // some code here
})();

# 变量

# 隐式声明全局变量

正常模式中，如果一个变量没有声明就赋值，默认是全局变量。严格模式禁止这种用法，全局变量必须显式声明。

'use strict'
v = 1
// ReferenceError: v is not defined

for (i = 0; i < 2; i++) {}
// ReferenceError: i is not defined

function f() { x = 123 }
f()
// ReferenceError: x is not defined

因此，严格模式下，变量都必须先声明再使用。

# 删除变量

严格模式下，无法使用 delete 操作符删除变量：

'use strict'
var x
delete x
// SyntaxError: Delete of an unqualified identifier in strict mode

# 对象

# 赋值只读属性

严格模式下，无法为只读属性赋值，如设置字符串的 length 属性：

'use strict'
'abc'.length = 5
// TypeError: Cannot assign to read only property 'length' of string 'abc'

因为 length 是只读属性，严格模式下不可写。正常模式下，改变 length 属性是无效的，但不会报错。

同样的，无法为一个只有取值器 getter 的属性赋值：

'use strict'
var obj = {
  get v() { return 1 }
}
obj.v = 2
// TypeError: Cannot set property v of #<Object> which has only a getter

obj.v 只有取值器，没有存值器，对它进行赋值就会报错。

# 删除不可配置属性

严格模式下，无法使用 delete 操作符删除不可配置的 (non-configurable) 属性：

'use strict'
var obj = Object.defineProperty({}, 'p', {
  value: 1,
  configurable: false
})
delete obj.p
// TypeError: Cannot delete property 'p' of #<Object>

# 扩展不可扩展对象

严格模式下，无法为不可扩展的 (nonextensible) 的对象添加属性：

'use strict'
var obj = {}
Object.preventExtensions(obj)
obj.v = 1
// TypeError: Cannot add property v, object is not extensible

# 函数

# 命名参数不唯一

正常模式下，可以通过 arguments 对象访问同名参数。严格模式下，这属于语法错误。

function sum(num, num) {
  'use strict'
  // do something
}
// SyntaxError: Duplicate parameter name not allowed in this context

# 不再追踪参数变化

正常模式下，修改命名参数的值会反映到 arguments 对象中。严格模式下，两者不再存在联动关系。

function f(a) {
  'use strict'
  a = 2
  return [a, arguments[0]]
}
f(1)
// 正常模式为: [2, 2]
// 严格模式为: [2, 1]

在非严格模式下，函数内部对命名参数的修改会改变 arguments[0] 的值，但在严格模式下，arguments[0] 仍然是传入的值。

# 访问 caller、caller、arguments 属性

arguments.callee 和 arguments.caller 是两个历史遗留的变量，从来没有标准化过，现在已经取消了。

# test

正常模式下调用它们没有什么作用，但是不会报错。

严格模式明确规定，函数内部不得使用 fn.caller、fn.callee、arguments 的属性，这意味着不能在函数内部得到调用栈。

'use strict'
var f = function () {
  f.caller
  f.arguments
  return arguments.callee
}
f()
// TypeError: 'caller', 'callee', and 'arguments' properties
// may not be accessed on strict mode functions or the arguments objects for calls to them

# eval

正常模式下，eval 语句的作用域取决于它所处的作用域。

严格模式下，eval 语句本身就是一个作用域，它在包含上下文中不再创建变量或函数。也就是说，eval 拥有自己的作用域。

function f() {
  'use strict'
  eval('var x = 10')
  console.log(x)
}
f()
// 正常模式下: 10
// 严格模式下: ReferenceError: x is not defined

在非严格模式下，以上代码会在函数 f 中创建一个局部变量 x，然后打印时会显示该变量的值。

在严格模式下，函数 f 中调用 eval() 不会创建变量 x，因此打印时会导致抛出 ReferenceError，因为 x 没有定义。

在 eval 中声明变量和函数属于 eval 作用域，随后就将被销毁。因此，以下代码可以运行：

'use strict'
var result = eval('var x = 10, y = 10; x + y')
console.log(result) // -> 20

在打印 result 时，尽管 x 和 y 已经被销毁，result 的值仍然是有效的。

# this

正常模式下，函数内部的 this 可能会指向全局对象。严格模式禁止这种用法，避免无意间创造全局变量。

function f() {
  'use strict'
  console.log(this)
}
f()
// 正常模式下: this 为 window
// 严格模式下: this 为 undefined

这种限制对于构造函数尤其有用。使用构造函数时，有时忘了加 new，这时 this 不再指向全局对象，而是报错。

function f() {
  'use strict'
  this.a = 1
}
f()
// TypeError: Cannot set property 'a' of undefined

严格模式下，函数直接调用时（不使用 new 调用），函数内部的 this 表示 undefined，因此可以用 call、apply 和bind 方法，将任意值绑定 在 this 上面。

'use strict'
function fun() {
  return this
}
fun()
// 正常模式: window, 严格模式: undefined
fun.call(2)
// 正常模式: Number {2}, 严格模式: 2
fun.call(true)
// 正常模式: Boolean {true}, 严格模式: true
fun.call(null)
// 正常模式: window, 严格模式: null
fun.call(undefined)
// 正常模式: window, 严格模式: undefined

上面代码中，可以把任意类型的值，绑定在 this 上面。

# 其他

# eval 与 arguments

严格模式已经明确禁止使用 eval 或者 arguments 作为标识符，也不允许读写它们的值。

'use strict'
var eval = 17
var arguments = 17
var obj = { set p(arguments) { } }
try { } catch (arguments) { }
function x(eval) { }
function arguments() { }
var y = function eval() { }
var f = new Function('arguments', "'use strict'; return 17;")
// SyntaxError: Unexpected eval or arguments in strict mode

# 禁止八进制字面量

正常模式下，整数的第一位如果是 0，表示这是八进制数，比如 0100 等于十进制的 64。

严格模式禁止这种表示法，整数第一位为 0 将报错。

'use strict'
var n = 0100
// SyntaxError: Octal literals are not allowed in strict mode

为此，ECMAScript 5 也修改了严格模式下 parseInt() 的行为，八进制字面量在严格模式下会被当作以 0 开头的十进制字面量。

'use strict'
parseInt('010') // 10

# 禁止 with 语句

正常模式下 with 语句能够改变解析标识符的路径。严格模式下，使用 with 语句将导致语法错误。

'use strict'
with(location) {
  console.log(href)
}
// SyntaxError: Strict mode code may not include a with statement
