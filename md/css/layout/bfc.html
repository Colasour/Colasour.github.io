# BFC

BFC (Block Formatting Context，块级格式化上下文)，它规定了内部的块级元素的布局方式，默认情况下只有根元素 body 一个块级上下文。

# 盒模型

MDN 每个盒子由四个部分（或称区域）组成，其效用由它们各自的边界所定义。与盒子的四个组成区域相对应，每个盒子有四个边界：内容边界 Content、内边距边界 Padding、边框边界 Border、外边框边界 Margin。

# 标准盒模型

浏览器的默认盒模型，CSS 属性为 box-sizing: content-box。

W3C 标准盒模型的 width 和 height 属性只包括 content 部分。

# IE 盒模型

IE 浏览器盒模型，CSS 属性为box-sizing: border-box。

IE 盒模型的 width 和 height 属性包括了 content、padding 和 border。

# 获取盒模型对应的宽高

  1. 通过 DOM style 样式获取

    element.style.width/hight;

    通过这种方式，只能获取行内样式，不能获取内嵌和外链的样式。

  2. 通过 DOM getComputedStyle() 方法

    window.getComputedStyle(element).width/height;

  3. 通过 DOM getBoundingClientRect() 方法

    element.getBoundingClientRect().width/height;

  4. 仅适于 IE 浏览器

    element.currentStyle.wdith/height;

# BFC 触发条件

  • 根元素，即 HTML 元素
  • float 的值不为 none
  • overflow 的值不为 visible
  • display 的值为 inline-block、table-cell、table-caption
  • position 的值为 absolute 或 fixed

# BFC 布局规则

  1. 内部的块级元素会在垂直方向，一个接一个地放置

  2. 块级元素垂直方向的距离由 margin 决定，属于同一个 BFC 的两个相邻的块级元素会发生 margin 合并，不属于同一个 BFC 的两个相邻的块级元素不会发生 margin 合并

  3. 每个元素的 margin box 的左边，与包含 border box 的左边相接触（对于从左到右的格式化，否则相反），即使存在浮动也是如此

  4. BFC 的区域不会与 float box 重叠

  5. BFC 就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素，外面的元素也不会影响到容器里面的子元素

  6. 计算 BFC 的高度时，浮动元素也参与计算

# 布局规则2：外边距折叠

MDN 块级元素的上外边距和下外边距有时会合并（或折叠）为一个外边距，其大小取其中的最大值，这种行为称为外边距重叠 (margin collapsping)。

下面列出了会发生外边距折叠的三种基本情况：

# 相邻元素之间

毗邻的两个元素之间的外边距会折叠（除非后一个元素需要清除之前的浮动）。

<div class="bor1"></div>
<div class="bor2"></div>

bor1 {
  width: 100px;
  height: 150px;
  margin-bottom: 20px;
  background-color: #FFCDD2;
}

.bor2 {
  width: 100px;
  height: 100px;
  margin-top: 50px;
  background-color: #448AFF;
}

可以看出，bor1 的 margin-bottom: 20px 塌陷在 bor2 的 margin-top: 50px 里。

# 父元素与其第一个或最后一个子元素之间

如果在父元素与其第一个子元素之间不存在边框、内边距、行内内容，也没有创建块级格式化上下文、或者清除浮动将两者的 margin-top 分开；

或者在父元素与其最后一个子元素之间不存在边框、内边距、行内内容、height、min-height、max-height 将两者的 margin-bottom 分开，那么这两对外边距之间会发生折叠。此时子元素的外边距会“溢出”到父元素的外面。

<div class="parent">
  <div class="child"></div>
</div>

.parent {
  width: 200px;
  background: green;
}

.child {
  height: 100px;
  margin-top: 10px;
  background: pink;
}

上面代码中，parent 的 height 是 100px，而不是 110px，因为 parent 和 child 在垂直方向上共一个 margin。

# 空的块级元素

如果一个块级元素中不包含任何内容，并且在其 margin-top 与 margin-bottom 之间没有边框、内边距、行内内容、height、min-height 将两者分开，则该元素的上下外边距会折叠。

一些需要注意的地方：

  • 上述情况的组合会产生更复杂的外边距折叠
  • 即使某一外边距为 0，这些规则仍然适用。因此就算父元素的外边距是 0，第一个或最后一个子元素的外边距仍然会“溢出”到父元素的外面
  • 如果参与折叠的外边距中包含负值，折叠后的外边距的值为最大的正边距与最小的负边距（即绝对值最大的负边距）的和
  • 如果所有参与折叠的外边距都为负，折叠后的外边距的值为最小的负边距的值。这一规则适用于相邻元素的嵌套元素

# 布局规则3：左边相接触

<div class="parent">
  <div class="child1"></div>
  <div class="child2"></div>
</div>

.parent {
  width: 150px;
  height: 40px;
  background-color: blue;
}

.child1 {
  float: left;
  width: 50px;
  height: 50px;
  background-color: pink;
}

.child2 {
  float: right;
  width: 50px;
  height: 50px;
  background-color: pink;
}

给这两个子 div 加浮动，浮动的结果，如果没有清除浮动的话，父 div 不会将下面两个 div 包裹，但还是在父 div 的范围之内。

左浮是子 div 的左边接触父 div 的 border box 的左边，右浮是子 div 接触父 div 的 border box 的右边，除非设置 margin 来撑开距离，否则一直是这个规则。

# 布局规则4：不与浮动盒重叠

<div class="aside"></div>
<div class="content">
  <div class="main"></div>
</div>

.aside {
  width: 100px;
  height: 150px;
  float: left;
  background: #f66;
}

.main {
  height: 200px;
  overflow: hidden;
  background: #fcc;
}

.content {
  width: 500px;
}

上面 aside 盒子有一个浮动属性，覆盖了 main 盒子的内容。main 盒子没有清除 aside 盒子的浮动，只做了一个动作，就是触发自身的 BFC，然后就不再被 aside 盒子覆盖了。所以，BFC 的区域不会与 float box 重叠。

# 布局规则5：隔离的独立容器

其实以上几个例子都体现了 BFC 布局规则5。

因为 BFC 内部的元素和外部的元素绝对不会互相影响，因此，当 BFC 外部存在浮动时，它不应该影响 BFC 内部盒的布局，BFC 会通过变窄，而不与浮动有重叠。

同样的，当 BFC 内部有浮动时，为了不影响外部元素的布局，BFC 计算高度时会包括浮动的高度，避免 margin 折叠也是这样的一个道理。

# BFC 应用

# 阻止元素被浮动元素覆盖

通过 BFC 布局规则4，触发生成 BFC，阻止元素被浮动元素覆盖，来实现自适应两栏布局。

<div class="main">
  <div class="left"></div>
  <div class="right"></div>
</div>

.main {
  height: 100vh;
  background: #fcc;
}

.left {
  float: left;
  width: 250px;
  height: inherit;
  background: #f66;
}

.right {
  overflow: hidden;
  height: inherit;
  background: #fdd;
}

# 包含浮动元素——清除内部浮动

<div class="parent">
  <div class="child"></div>
  <div class="child"></div>
</div>

.parent {
  border: 5px solid #fcc;
}

.child {
  border: 5px solid #f66;
  width: 100px;
  height: 100px;
  float: left;
}

根据 BFC 布局规则4，为达到清除内部浮动，可以触发 parent 生成 BFC，那么 parent 在计算高度时，parent 内部的浮动元素child 也会参与计算。

.parent {
  overflow: hidden;
}

# 分属不同的 BFC 时可以阻止 margin 折叠

<p>Haha</p>
<p>Hehe</p>

p {
  color: #f55;
  background: #fcc;
  width: 200px;
  line-height: 100px;
  text-align: center;
  margin: 100px;
}

可以在 p 标签外面包裹一层容器，并触发该容器生成一个 BFC。那么两个 p 标签便不属于同一个 BFC，就不会发生外边距重叠了。

<p>Haha</p>
<div class="wrap">
  <p>Hehe</p>
</div>

.wrap {
  overflow: hidden;
}

# BFC 与 Layout

IE 中存在 Layout，Layout 与 BFC 基本是等价的。为了处理 IE 的兼容性，在需要触发 BFC 时，除了需要用触发条件中的 CSS 属性来触发 BFC，还需要针对 IE 浏览器使用 zoom: 1 来触发 IE 浏览器的 Layout。
